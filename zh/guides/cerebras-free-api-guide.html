方法一：用Python SDK（推荐）

pip install --upgrade cerebras_cloud_sdk import os from cerebras.cloud.sdk import Cerebras client = Cerebras( api_key=os.environ.get("CEREBRAS_API_KEY"), ) response = client.chat.completions.create( messages=[ {"role": "user", "content": "用Python写一个快速排序"} ], model="gpt-oss-120b", ) print(response.choices[0].message.content)

方法二：用cURL直接调用

curl https://api.cerebras.ai/v1/chat/completions \ -H "Content-Type: application/json" \ -H "Authorization: Bearer $CEREBRAS_API_KEY" \ -d '{ "model": "gpt-oss-120b", "messages": [ {"role": "user", "content": "用Python写一个快速排序"} ] }'

因为Cerebras完全兼容OpenAI API格式，你可以直接用OpenAI的Python库，只需要改两行：

from openai import OpenAI client = OpenAI( api_key=os.environ.get("CEREBRAS_API_KEY"), base_url="https://api.cerebras.ai/v1", ) response = client.chat.completions.create( model="gpt-oss-120b", messages=[{"role": "user", "content": "Hello!"}] )

⚠️ 新手最容易踩的5个坑

🚫 坑1：忘记设置代理

Cerebras的API地址（api.cerebras.ai）在国内无法直接访问。解决方案有两个：一是用代理软件，二是用Cloudflare Workers搭建中转（搜索"Cerebras CF中转"能找到现成方案）。

🚫 坑2：RPM 5不代表慢

每分钟5个请求听起来很低，但Cerebras单次响应极快（GPT OSS 3000 tokens/s），所以5个请求的实际吞吐量远超其他平台。关键是一次请求处理更多内容，而不是频繁发短请求。

🚫 坑3：API Key只出现一次

创建API Key时只显示一次，关掉后就看不到了。务必立刻复制保存到安全的地方（比如密码管理器）。

🚫 坑4：没设max_completion_tokens导致超限

如果不指定max_completion_tokens，Cerebras会预估最大序列长度来扣减token配额，可能导致很快用完100万token。建议始终设置合理的max_completion_tokens值。

🚫 坑5：混淆免费层和付费层

免费层RPM只有5，如果你要做高频调用（比如批量处理），可以考虑升级到Developer层（最低$10），RPM提升到1000。
