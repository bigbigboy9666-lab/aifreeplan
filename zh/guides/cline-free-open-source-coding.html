一个完全开源、不锁模型的编程助手

用过 GitHub Copilot 的人都有一个痛点：每月 10 刀，模型就那几个，想换还得看脸色。Cursor 更狠，20 刀一个月。直到我发现了 Cline——完全开源，Apache 2.0 协议随便商用，关键是不锁模型。Claude、GPT、Gemini、DeepSeek，200 多个模型随你切。自备 API Key 就行，没有订阅费。

Cline 到底是什么东西

Cline 是一个开源 AI 编程助手，能跑在 VS Code、JetBrains 和命令行里。和那些商业工具比，它有几个根本性的不同：代码在 GitHub 上完全公开，Apache 2.0 协议意味着你可以商用、改、再分发，没人管你。模型方面，你可以接 Claude、GPT-4o、Gemini、DeepSeek、Qwen，通过 OpenRouter 能用 200 多个模型，完全不绑定。费用方面只付 API 调用的钱，用 DeepSeek 的话编程任务每小时大概 1 到 5 美分。GitHub 上星标破万，社区更新频率很高。

和 Copilot、Cursor 比到底差在哪

价格上，Cline 免费（自备 Key），Copilot 每月 10 刀，Cursor 每月 20 刀，Windsurf 每月 15 刀。模型数量上，Cline 能接 200 多个，其他几家最多给你几个选项。开源方面只有 Cline 是，其他三个都是闭源商业产品。IDE 支持上 Cline 覆盖 VS Code、JetBrains 和命令行，Copilot 也能用这两个 IDE，但 Cursor 和 Windsurf 只能用自家的。核心区别在于：Cline 的"免费"是真免费，没有功能阉割，没有次数上限，没有排队。唯一的成本是 API 调用费，而这个费用你自己可以控制——选便宜的模型就行。

最聪明的设计：Plan/Act 双模式

Cline 有两个工作模式，这是我觉得它最聪明的地方。Plan 模式下，Cline 先分析你的需求，制定修改计划，列出要改哪些文件、怎么改，你确认了才执行。Act 模式下它就直接干活了，自动创建修改文件、跑终端命令、修 bug。我的用法是先 Plan 让它出方案，确认没问题再切 Act 执行。这样更可控，不容易翻车。

Kanban 看板：多个任务同时跑

Cline 有个独特的看板功能，可以同时启动多个编程任务，后台并行处理。比如"重构用户模块""修复登录 bug""写单元测试"三件事同时跑，你能在看板上看到每个任务的进度。大型项目的时候这个功能特别香。

怎么把成本压到最低

既然要自备 API Key，那怎么花最少的钱？最便宜的方案是 DeepSeek，V4 API 价格极低，编程任务每小时 1 到 5 美分，中文编程理解也不错。性价比最高的是 Claude Sonnet，每百万输入 token 3 刀，编程能力在所有模型里排前列，重度使用每天大概 0.5 到 2 刀。最灵活的是 OpenRouter，一个 Key 接 200 多个模型，可以按任务选模型，价格透明。预算有限的话，DeepSeek 加 Cline 是目前最便宜的 AI 编程方案，比 GitHub Copilot 免费版还划算——Copilot 有 2000 次限制，Cline 无限制。

想了解更多免费编程工具，可以看看 Freebuff 免费编程 Agent 和 AtomCode CodingPlan 攻略。

新手最容易踩的坑

Cline 本身免费开源没错，但你得有自己的 API Key，没有 Key 它就是个空壳。很多人装完发现用不了，原因就在这里。另外 API 费用是按 token 计的，复杂任务可能吃掉很多 token，建议设个上限防止意外账单。还有人问能不能和 Cursor 或 Windsurf 一起用——可以，Cline 是 VS Code 插件，跟它们不冲突，不同场景用不同工具就行。

❓ 常见问题

Cline 真的完全免费吗？

是的。Cline 本身完全免费、开源、无限制。你需要自备 AI 模型的 API Key，这部分有成本。

Cline 和 GitHub Copilot 哪个好？

Cline 免费无限制但需要自备 Key，Copilot 免费版有 2000 次限制。Cline 更划算。

Cline 支持哪些 AI 模型？

通过 OpenRouter 可以用 200+ 模型，包括 Claude、GPT、Gemini、DeepSeek、Qwen 等。

Cline 安全吗？代码会泄露吗？

Cline 本身是开源的，代码可审计。安全性取决于你选择的 API 提供商。

新手能用 Cline 吗？

能，但需要一点配置（安装扩展 + 填 API Key）。如果不想配置，用 Copilot 免费版更简单。
