💻 Your First API Call

Method 1: Python SDK (Recommended)

pip install --upgrade cerebras_cloud_sdk import os from cerebras.cloud.sdk import Cerebras client = Cerebras( api_key=os.environ.get("CEREBRAS_API_KEY"), ) response = client.chat.completions.create( messages=[ {"role": "user", "content": "Write a quick sort in Python"} ], model="gpt-oss-120b", ) print(response.choices[0].message.content)

Method 2: Direct cURL

curl https://api.cerebras.ai/v1/chat/completions \ -H "Content-Type: application/json" \ -H "Authorization: Bearer $CEREBRAS_API_KEY" \ -d '{ "model": "gpt-oss-120b", "messages": [ {"role": "user", "content": "Write a quick sort in Python"} ] }'

Since Cerebras is fully OpenAI API-compatible, you can use the OpenAI Python library directly with just two changes:

⚠️ 5 Common Mistakes Beginners Make

🚫 Mistake 1: Forgetting about the proxy

The Cerebras API endpoint (api.cerebras.ai) is not directly accessible from mainland China. Solutions: use a proxy, or set up a Cloudflare Workers relay (search "Cerebras CF relay" for ready-made solutions).

🚫 Mistake 2: Thinking RPM 5 is slow

5 requests per minute sounds low, but each Cerebras response is incredibly fast (GPT OSS hits 3000 tokens/s). So 5 requests actually process far more tokens than other platforms. The key is to send fewer, bigger requests rather than many small ones.

🚫 Mistake 3: Losing your API Key

Your API key only displays once when created. Copy and save it immediately (e.g., in a password manager).

🚫 Mistake 4: Not setting max_completion_tokens

If you don't specify max_completion_tokens, Cerebras estimates using the maximum sequence length, which can drain your 1 million token quota faster than expected. Always set a reasonable max_completion_tokens value.

🚫 Mistake 5: Confusing Free and Paid tiers

The free tier has RPM 5. If you need high-frequency calls (e.g., batch processing), consider upgrading to the Developer tier (starting at $10), which bumps RPM to 1000.
