In a Nutshell

GLM-5.1 ranked first among open-source models in the newly released Coding Agent benchmark by Artificial Analysis, second only to Opus 4.7. What Makes GLM-5.1 So Strong

Artificial Analysis's Coding Agent Index tested three benchmarks: SWE-Bench-Pro-Hard-AA, Terminal-Bench v2, and SWE-Atlas-QnA. The top closed-source model is Opus 4.7 (in a Cursor CLI environment), while the top open-source model is GLM-5.1 (in a Claude Code environment).

This evaluation wasn't cherry-picking easy tasks. The SWE-Bench series is currently the toughest benchmark for coding ability, Terminal-Bench tests terminal operation capabilities, and SWE-Atlas-QnA assesses code comprehension. Ranking first across all three tests indicates that GLM-5.1 is far beyond just writing "hello world".

On the same day, Kilo Code released a programming-specific leaderboard, where GLM-5.1 also ranked first among domestic models. Two independent evaluation bodies arriving at the same conclusion on the same day feels a bit too deliberate to be a coincidence.

    What Does the CodingPlan Actually Include?

    AtomCode's CodingPlan has two tiers, both using a 5-hour rolling window metering system (usage resets every 5 hours) with a 30-day validity period.

    Lite Tier - Light Experience

    Suitable for beginners just trying things out. Supported models:

  • DeepSeek-V4-Flash
  • Qwen3.6-35B-A3B
  • MiniMax-M2.7

    • Basic usage limits, sufficient for writing small projects.

    Pro Tier - The Developer Workhorse

    The quota is approximately 5 times that of the Lite tier, with a impressive lineup of supported models:

  • GLM-5.1 (free for a limited time)
  • DeepSeek-V4-Flash
  • Qwen/Qwen3 full series (including 235B-A22B, 32B, 30B-A3B, etc.)
  • Qwen-VL multimodal models

    • According to information from the AtomCode official page, the Pro tier supports even more models: DeepSeek-V3.2, DeepSeek-R1, Qwen3.5-35B-A3B, etc. The entire model lineup covers half of China's mainstream open-source large models.

    How to Claim It - Step by Step

    Step 1: Install the Client

    For Mac or Linux:

    ```bash

    curl -fsSL https://atomgit.com/atomgit_atomcode/atomcode/releases/download/v4.21.2/install.sh | sh

    ```

    For Windows PowerShell:

    ```powershell

    irm https://atomgit.com/atomgit_atomcode/atomcode/releases/download/v4.21.2/install.ps1 | iex

    ```

    Step 2: Log In

    Enter `atomcode` in the command line to start the client, and log in by scanning the WeChat QR code. If you don't have an account, register first at atomgit.com.

    Step 3: Claim Your Quota

    In the client, enter `/codingplan` and click to claim. The welcome screen on first launch also has a "Set up CodingPlan free quota" button—the same entry point.

    Important Notes

  • Daily free quotas are limited. Articles from Official Accounts in Sogou Search mention a daily limit of 2,000 Pro packages, while another article says the limit is 800. The exact number may fluctuate; the point is, act fast.
  • If you see the message "Today's application quota is full," you'll have to try again the next day.
  • It can only be used within the AtomCode client; third-party API calls are not supported. This route doesn't work for trying to call it via `curl` or Python scripts.
